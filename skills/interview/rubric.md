# 面试评分标准（rubric.md）

## 通用评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 工作经历匹配 | 30% | 项目经验与岗位需求的匹配程度、证据具体度 |
| 专业能力 | 40% | 技术深度/业务理解、问题解决能力、系统思考 |
| 软技能 | 30% | 表达清晰度、逻辑结构、追问承压、自我认知 |

## 现场表现评分细项

| 评分项 | 说明 |
|--------|------|
| 岗位匹配度 | 回答内容与岗位核心要求的相关性 |
| 证据具体度 | 是否给出了具体场景、数据、结果 |
| 逻辑清晰度 | 回答结构是否清晰，论证是否自洽 |
| 表达自然度 | 是否像真实经历的口述，还是背诵/编造感 |
| 追问承压度 | 面对追问是否稳定、能否补充细节 |
| 风险控制度 | 是否暴露了重大风险（造假、逻辑矛盾、回避） |

## 岗位类型权重调整

### 技术岗
- 专业能力权重提升至 50%
- 重点：系统设计、代码能力、技术决策、性能优化
- 追问方向：技术选型原因、性能数据、故障处理

### 产品岗
- 工作经历权重提升至 35%
- 重点：用户洞察、数据驱动、需求优先级、跨部门推动
- 追问方向：DAU/留存/转化等核心指标、需求取舍决策

### 运营/销售
- 软技能权重提升至 35%
- 重点：目标达成、资源撬动、客户管理、数据复盘
- 追问方向：具体 GMV/ROI 数据、客户案例、失败复盘

### 管理岗
- 软技能权重提升至 40%
- 重点：团队搭建、绩效管理、向上汇报、资源争取、战略拆解
- 追问方向：团队规模、淘汰决策、跨部门冲突、目标拆解逻辑

## 评分等级

| 分数段 | 含义 |
|--------|------|
| 90-100 | 证据清楚、业务感强、表达自然、可承受追问 |
| 70-89 | 结构可用，但有细节或证据漏洞 |
| 40-69 | 能回答但泛泛，缺关键事实或岗位相关性弱 |
| 0-39 | 未回答、请求提示、明显造假或严重跑题 |
